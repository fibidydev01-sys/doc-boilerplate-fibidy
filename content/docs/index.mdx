Welcome. These docs are your complete guide to setting up, deploying, and extending the boilerplate.

## What You Get

A production-ready Next.js 16 + Supabase boilerplate. Swap the `.env` file and your logo, and the application is ready to ship. No need to rewrite auth, dashboards, commerce integration, or email delivery from scratch.

**Included out of the box:**

- Complete auth flows — email/password, magic link, Google OAuth, forgot/reset password
- Role-based access control with permission matrix and wildcard support
- Commerce module — Lemon Squeezy integration (products, orders, subscriptions, customers, checkout, webhooks)
- Transactional auth emails via Resend + React Email templates (auto-rebrand)
- Supabase Send Email Hook — replaces Supabase's limited default SMTP
- Dashboard with sidebar, mobile bottom nav, and admin panel
- Built-in i18n (Indonesian + English) with full type safety
- Append-only activity logging for audit trails
- AES-256-GCM encryption for sensitive data (API keys, webhook secrets)
- PWA manifest + service worker ready
- 8 module slots (admin + commerce fully built, 6 placeholders ready to extend)

**Tech stack at a glance:** Next.js 16 · React 19 · TypeScript · Supabase · Tailwind v4 · Radix UI + shadcn/ui · Zustand · React Hook Form + Zod v4 · Resend · Sonner · pnpm.

Full details in [Tech Stack](/docs/tech-stack).

## Documentation Conventions

- **✅** = already implemented, ready to use
- **⏳** = placeholder / extension point (you implement it)
- **⚠️** = warning, read carefully
- **🔒** = security-sensitive, don't leak
- Code blocks starting with `#` are comments, not commands
- File paths use the format `src/modules/commerce/...` — relative to project root
